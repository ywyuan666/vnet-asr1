# 耳机语音指令识别系统 - 项目说明文档

**基于 WeNet U2++ Conformer 的端到端中文语音识别系统**

---

## 项目概述

本项目实现了一个面向蓝牙耳机应用场景的端到端中文语音指令识别系统。系统能够识别用户发出的中文语音命令（如「下一首」「打开降噪」「接听电话」「调高音量」等），并据此触发相应的耳机功能。

项目采用业界先进的 **WeNet 框架**和 **U2++ 架构**，包含 Conformer 编码器、CTC/Attention 联合训练、双向注意力解码器（bitransformer）以及 Attention Rescoring 重打分机制，支持流式与非流式统一识别。

---

## 核心技术架构

### 1. 端到端 ASR 方案

系统采用端到端（End-to-End）语音识别方案，直接从语音波形映射到汉字输出，无需传统方案中的 HMM-GMM 声学模型、发音词典和语言模型的独立训练与集成，极大简化了系统复杂度。

### 2. Conformer 编码器

Conformer 是结合了 Transformer 自注意力机制和卷积神经网络的混合架构：

- **多头自注意力（MHSA）**：捕捉全局上下文依赖，建模长距离声学特征关系
- **卷积模块**：提取局部时序特征，增强相邻帧之间的细粒度建模能力
- **Macaron 结构**：采用"三明治"式的前馈网络布局，提升特征表达能力

### 3. CTC 与 Attention 联合训练

**CTC（Connectionist Temporal Classification）分支：**
- 非自回归解码，速度快
- 引入 blank 符号自动学习帧-字符对齐
- 适合快速一遍解码

**Attention 解码器：**
- 自回归生成文字序列
- 建模字符间依赖关系，精度高
- 通过交叉注意力机制聚焦相关声学帧

联合训练使两者优势互补，在速度与精度之间取得平衡。

### 4. U2++ 双向解码器

U2++ 是 U2（Unified Streaming and Non-streaming）架构的增强版本，核心创新在于：

**双向注意力解码器：**
- **L2R（Left-to-Right）解码器**：从左到右生成文字
- **R2L（Right-to-Left）解码器**：从右到左生成文字
- 双向联合建模显著提升边界识别准确率和整体 CER 指标

**流式与非流式统一：**
- 通过动态 chunk 训练和因果卷积
- 同一模型既支持整句识别（非流式，最高精度）
- 也支持实时识别（流式，低延迟）

### 5. Attention Rescoring

两遍解码策略：
1. **第一遍**：CTC 快速生成 N-best 候选列表
2. **第二遍**：Attention 解码器对候选进行重打分，选择最优结果

兼顾 CTC 的解码速度与 Attention 的识别精度。

### 6. 数据增强策略

**速度扰动（Speed Perturbation）：**
- 对训练音频进行 0.9x、1.0x、1.1x 倍速变换
- 扩充数据规模，增强对不同说话速度的鲁棒性

**频谱掩蔽（SpecAugment）：**
- 在 Fbank 特征上随机遮挡时间段和频率段
- 防止过拟合，提升泛化能力

## 系统工作流程

```
输入：麦克风 / 音频文件
   ↓
Fbank 特征提取（80 维梅尔滤波器组）
   ↓
CMVN 归一化（零均值单位方差）
   ↓
Conformer 编码器（多层 Conformer 块）
   ↓
   ├──→ CTC 分支 → N-best 候选列表
   │
   └──→ 双向 Attention 解码器（L2R + R2L）
         ↓
      Attention Rescoring 重打分
         ↓
输出：识别文字
```

## 项目实现细节

### 数据集构建

由于缺少现成的中文耳机指令语音数据集，项目采用 **TTS 合成方案**：

- 使用 Microsoft edge-tts 在线神经网络语音合成服务
- 采用 6 个不同发音人（包含男声、女声、不同口音）
- 每条指令合成多个语速版本（通过 `--repeat` 参数控制）
- 配合训练时的速度扰动和频谱掩蔽，构建可训练的小型数据集

### 模型训练配置

**编码器配置：**
- 隐藏维度：144
- 注意力头数：4
- Conformer 层数：6
- 前馈维度：1024

**解码器配置：**
- L2R 解码器层数：3
- R2L 解码器层数：3

**损失权重：**
- CTC 损失权重：0.3
- R2L 解码器权重：0.3

**训练参数：**
- 优化器：Adam，学习率 0.001
- 学习率调度：Warmup LR（warmup_steps=500）
- 最大训练轮数：60
- 批量大小：16

### 评测指标

使用 **CER（Character Error Rate，字错误率）** 作为主要评测指标：

```
CER = (替换字符数 + 删除字符数 + 插入字符数) / 参考文本总字符数 × 100%
```

CER 越低表示识别精度越高。

### 解码模式对比

| 解码模式 | 特点 | 速度 | 精度 |
| --- | --- | --- | --- |
| `ctc_greedy_search` | CTC 贪心搜索 | 最快 | 一般 |
| `ctc_prefix_beam_search` | CTC 前缀束搜索 | 快 | 较好 |
| `attention` | 纯注意力解码 | 慢 | 好 |
| `attention_rescoring` | CTC N-best + 注意力重打分 | 中等 | **最好** |

推荐使用 `attention_rescoring` 模式以获得最佳识别效果。

## 使用流程

### 一键运行

```bash
# Windows
.\run.ps1

# Linux / Mac
bash run.sh
```

脚本将依次执行以下阶段：

1. **Stage 0**：TTS 合成语音数据集
2. **Stage 1**：准备 WeNet 数据格式
3. **Stage 2**：计算 CMVN 统计量
4. **Stage 3**：训练 U2++ Conformer 模型
5. **Stage 4**：解码测试集并计算 CER
6. **Stage 5**：单条音频识别演示

### 分步执行

**生成数据集：**
```bash
python local/generate_corpus.py --out data/audio --repeat 2
```

**准备训练数据：**
```bash
python local/prepare_data.py --audio_dir data/audio --out_dir data
```

**计算 CMVN：**
```bash
python tools/make_cmvn.py --data_list data/train/data.list --out data/train/global_cmvn
```

**训练模型：**
```bash
python -m wenet.bin.train \
  --config conf/train_u2pp_conformer.yaml \
  --data_type raw \
  --train_data data/train/data.list \
  --cv_data data/dev/data.list \
  --model_dir exp/u2pp_conformer \
  --cmvn data/train/global_cmvn \
  --num_workers 2
```

**评测模型：**
```bash
python recognize.py \
  --config exp/u2pp_conformer/train.yaml \
  --checkpoint exp/u2pp_conformer/final.pt \
  --test_data data/test/data.list \
  --dict data/dict/units.txt \
  --mode attention_rescoring
```

**实时识别：**
```bash
# 识别 wav 文件
python infer_demo.py --checkpoint exp/u2pp_conformer/final.pt \
  --dict data/dict/units.txt --wav test.wav

# 麦克风录音识别
python infer_demo.py --checkpoint exp/u2pp_conformer/final.pt \
  --dict data/dict/units.txt --mic 3
```

## 常见问题

### Q1：没有 GPU 能否训练？

**A**：可以。项目数据集规模较小，配置中 batch size 已针对 CPU 优化，使用 CPU 也能完成训练，只是耗时相对较长（约 1-2 小时）。

### Q2：CER 偏高怎么办？

**A**：合成数据 + 小模型架构下 CER 偏高属正常现象。改进方向：
- 调大 `--repeat` 参数扩充数据规模
- 增加 `max_epoch` 延长训练时间
- 精简指令集降低任务难度
- 收集真实录音数据替代合成数据

### Q3：找不到 `wenet.bin.train` 模块？

**A**：`pip install wenet` 仅安装推理库。训练入口需要从源码安装：
```bash
git clone https://github.com/wenet-e2e/wenet.git
cd wenet
pip install -e .
```

### Q4：ffmpeg 相关报错？

**A**：确认 ffmpeg 已正确安装并添加到系统 PATH。安装后需重新打开终端使环境变量生效。

### Q5：配置 yaml 字段报错？

**A**：不同 WeNet 版本配置字段略有差异。按报错提示调整 `conf/train_u2pp_conformer.yaml` 中的对应字段，核心结构（conformer / bitransformer / ctc）保持不变即可。
