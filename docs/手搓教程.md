### 0.4 装 Git（下载 WeNet 代码要用）

- 启动页点 `Open`，选中 `my-anker-asr`，打开。

### 1.3 创建"虚拟环境"（venv）—— 非常重要

> 虚拟环境 = 给这个项目单独开一个干净的 Python "小房间"，装的库不污染系统。

**方式一：用 PowerShell 手动建（VSCode/PyCharm 通用，最稳）**

在 IDE 里打开终端：
- VSCode：菜单 `Terminal → New Terminal`。
- PyCharm：左下角点 `Terminal` 标签。

确认终端开头路径是你的项目文件夹，然后执行：
```powershell
python -m venv .venv
```
执行完项目里会多出一个 `.venv` 文件夹。接着**激活**它：
```powershell
.\.venv\Scripts\Activate.ps1
```

> ⚠️ 如果报红字 `无法加载…因为在此系统上禁止运行脚本`，执行下面这条解除限制，再重新激活：
> ```powershell
> Set-ExecutionPolicy -Scope CurrentUser RemoteSigned
> ```
> 激活成功的标志：终端每行最前面出现 `(.venv)`。

**方式二：PyCharm 图形界面建**
- `File → Settings → Project → Python Interpreter` → 右上角齿轮 `Add` → 选 `Virtualenv Environment → New` → 确定。PyCharm 会自动建好并选用。

### 1.4 让 IDE 用上这个虚拟环境

**VSCode**：按 `Ctrl + Shift + P`，输入 `Python: Select Interpreter`，回车，选列表里带 `.venv` 字样的那个。

**PyCharm**：如果用方式一手动建的，去 `Settings → Project → Python Interpreter → Add → Existing` → 选 `.venv\Scripts\python.exe`。

✅ **检查点**：终端前面有 `(.venv)`，且 IDE 右下角/解释器设置里显示的是 `.venv` 里的 Python → 成功。

## 第 3 部分：先理解模型（跑教学版 U2++）

> 为什么先做这个？因为它**不依赖 WeNet、不依赖数据**，能让你立刻"看见" U2++ 内部长啥样，
> 对写报告/答辩帮助极大。先尝甜头，建立信心。

### 3.1 创建 `model/u2pp_conformer_min.py`

新建文件夹 `model`，里面建文件 `u2pp_conformer_min.py`，内容用桌面标准答案。

**你要理解的 4 个部分（答辩会问）**：
1. `ConformerBlock`：½前馈 → 多头自注意力 → 卷积模块 → ½前馈（这就是 Conformer 的"三明治"结构）。
2. `ctc_fc` + `F.ctc_loss`：CTC 分支（快速解码）。
3. `decoder_l2r`：左→右注意力解码器。
4. `decoder_r2l`：右→左注意力解码器 ← **这就是 U2++ 比 U2 多出来的"++"**。

### 3.2 运行它

**用终端跑**（推荐）：
```powershell
python model/u2pp_conformer_min.py
```

**用 IDE 按钮跑**：
- VSCode：打开该文件，点右上角 ▷ 三角形 `Run Python File`。
- PyCharm：打开该文件，右键编辑区 → `Run 'u2pp_conformer_min'`。

✅ **检查点**：终端打印类似下面，就说明模型结构完全正确（这是真实运行结果）：
```
教学版 U2++ Conformer 前向测试
参数量          : 7.97 M
编码器输入      : (2, 100, 80)  (B,T,80)
编码器输出      : (2, 24, 144)  (4倍下采样)
CTC 损失        : 8.36
L2R 注意力损失  : 3.93
R2L 注意力损失  : 3.71  ← U2++ 双向解码
联合总损失      : 5.21
结构验证通过 ✅
```
看懂了：100 帧经过 Conformer 前端下采样 4 倍变成 24 帧；CTC 和左右两个解码器各算了一份损失，最后合成联合损失。**这就是 U2++ 的训练逻辑。**

## 第 4 部分：生成语音数据集

> 小白没有录音数据，我们用微软 TTS（文字转语音）自动合成"安克耳机指令"音频。
> 多个发音人 + 多语速 = 自带数据多样性。

### 4.1 创建 `local/generate_anker_corpus.py`

新建文件夹 `local`，里面建 `generate_anker_corpus.py`，内容用桌面标准答案。

**你要看懂的地方**：
- 顶部 `ANKER_COMMANDS`：14 条中文指令（下一首、打开降噪、接听电话…）。**这里就是模型要学会识别的目标，你可以随意增删。**
- `VOICES`：6 个不同发音人（男声/女声/口音），相当于不同说话人。
- 它会把每条指令用每个发音人念一遍，存成 16kHz 单声道 wav，并写一份 `metadata.tsv`（音频路径↹文字）。

### 4.2 运行

```powershell
python local/generate_anker_corpus.py --out data/audio --repeat 2
```
> `--repeat 2` 表示重复 2 轮（换语速），数据更多。想更多就调大，比如 `--repeat 4`。
> 这一步需要联网（要访问微软语音服务）。

✅ **检查点**：
- 终端滚动打印 `[1] next_Xiaoxiao_r0 -> 下一首` 这样的行，最后显示"完成！共生成 XXX 条音频"。
- 打开 `data/audio` 文件夹，看到一堆 `.wav` 文件和一个 `metadata.tsv`。
- 随便双击播放一个 wav，能听到人声念指令 → 完美。

> ❌ 如果报 `未检测到 ffmpeg` → 回第 0.3 装 ffmpeg 并**重开终端**。
> ❌ 如果报网络/超时 → 检查网络，或重跑（已生成的会覆盖，不影响）。

## 第 5 部分：准备 WeNet 数据格式

> WeNet 训练要求特定格式：一份 `data.list`（每行一个 JSON）+ 一份字典 `units.txt`。
> 这一步把上一步的音频整理成它要的样子，并自动划分 训练/验证/测试 集。

### 5.1 创建 `local/prepare_data.py`

在 `local` 文件夹里建 `prepare_data.py`，内容用桌面标准答案。

**它做了什么**：
- 读 `metadata.tsv`，按 8:1:1 划分成 train/dev/test。
- 每个集合写一份 `data.list`，每行形如：
  `{"key": "next_Xiaoxiao_r0", "wav": "C:/.../next_Xiaoxiao_r0.wav", "txt": "下一首"}`
- 扫描所有文字里出现的汉字，生成字典 `data/dict/units.txt`，格式 `字 编号`，
  其中固定包含 `<blank> 0`（CTC 空白符）、`<unk> 1`（未知字）、`<sos/eos>`（句子起止符）。

### 5.2 运行

```powershell
python local/prepare_data.py --audio_dir data/audio --out_dir data
```

✅ **检查点**：
- 终端打印 `train: xxx 条`、`dev: xx 条`、`test: xx 条`、`字典大小(含特殊符): xx`。
- 出现这些文件：`data/train/data.list`、`data/dev/data.list`、`data/test/data.list`、`data/dict/units.txt`。
- 用 IDE 打开 `data/train/data.list` 看一眼，每行是一个 JSON，路径和文字对得上 → 成功。

## 第 8 部分：写训练配置 + 开始训练

### 8.1 创建 `conf/train_u2pp_conformer.yaml`

新建文件夹 `conf`，里面建 `train_u2pp_conformer.yaml`，内容用桌面标准答案。

**这是整个项目的"大脑配置"，你必须能讲出这几块**：
- `encoder: conformer` + `causal: true` + `use_dynamic_chunk: true` → Conformer 编码器，支持流式。
- `decoder: bitransformer` + `r_num_blocks: 3` → **双向解码器，U2++ 的核心**。
- `model_conf`：`ctc_weight: 0.3`（CTC 占比）、`reverse_weight: 0.3`（右→左解码器占比）。
- `dataset_conf`：`speed_perturb`（速度扰动）、`spec_aug`（频谱掩蔽）两种数据增强。
- `max_epoch: 60`：训练轮数（数据少可以多训几轮）。

### 8.2 开始训练

复制下面这一整段到终端（注意每行末尾的 `` ` `` 是 PowerShell 的换行符，别漏）：
```powershell
python -m wenet.bin.train `
  --config conf/train_u2pp_conformer.yaml `
  --data_type raw `
  --symbol_table data/dict/units.txt `
  --train_data data/train/data.list `
  --cv_data data/dev/data.list `
  --model_dir exp/u2pp_conformer `
  --cmvn data/train/global_cmvn `
  --num_workers 2 `
  --pin_memory
```

**会发生什么**：终端开始一轮轮（epoch）打印 loss，数字总体往下降。没显卡用 CPU 也能跑，慢一点，泡杯茶等它。

✅ **检查点**：
- 看到类似 `Epoch 0 ... loss 5.2 ...`、`Epoch 1 ... loss 4.1 ...`，loss 在下降。
- `exp/u2pp_conformer/` 文件夹里陆续出现 `0.pt`、`1.pt`… 以及训练用的 `train.yaml`。
- 训练跑完（到 max_epoch）会有 `final.pt`（或用最后一个 `*.pt` 也行）。

> ❌ 报"unexpected key / unknown argument"之类 → 是你的 WeNet 版本字段略有不同。
>   按报错提示，去 `conf/train_u2pp_conformer.yaml` 删掉/改掉它指出的那一行即可，
>   **核心结构（conformer / bitransformer / ctc）不要动**。
> ❌ 训练太慢 → 把 yaml 里 `max_epoch` 改小（如 30）、`num_blocks` 改小（如 4）先跑通流程。
> ❌ 想中途停 → 终端按 `Ctrl + C`，已保存的 `*.pt` 还能用。

## 第 9 部分：解码评测（算 CER）

> 用训练好的模型在测试集上识别，并算出 **CER 字错误率**（越低越好）。

### 9.1 创建 `recognize.py`

在项目根目录建 `recognize.py`，内容用桌面标准答案。
它内部调用 WeNet 最稳的解码入口 `wenet.bin.recognize`，再自动对照标注算 CER。

### 9.2 运行

```powershell
python recognize.py `
  --config exp/u2pp_conformer/train.yaml `
  --checkpoint exp/u2pp_conformer/final.pt `
  --test_data data/test/data.list `
  --dict data/dict/units.txt `
  --mode attention_rescoring
```
> 如果没有 `final.pt`，把 `--checkpoint` 换成 `exp/u2pp_conformer/` 里序号最大的那个 `.pt`。

✅ **检查点**：终端逐条打印 `✅/❌ 参考: 打开降噪 | 识别: 打开降噪`，最后给出 `CER (字错误率) = xx.xx%`。

**做实验对比（写报告用）**：把 `--mode` 依次换成下面 4 个各跑一次，把 CER 填进 `docs/答辩大纲.md` 的表格里：
`ctc_greedy_search` / `ctc_prefix_beam_search` / `attention` / `attention_rescoring`。
通常 `attention_rescoring` 最准。

> 💡 数据是合成的、模型也调小了，CER 偏高很正常。报告里如实写"受数据规模/模型规模限制"，
>   重点是**流程完整、方法正确**。想降 CER：第 4 步 `--repeat` 调大造更多数据、yaml 里 `max_epoch` 调大。

## 第 11 部分：一键脚本（收尾）

> 前面你已经一步步手动跑通了。最后把这些命令打包成一键脚本，方便重跑和答辩演示。

在项目根目录建 `run.ps1`（Windows）和 `run.sh`（如果用 WSL/Linux），内容用桌面标准答案。
以后想从某一步重跑，直接：
```powershell
.\run.ps1 -Stage 3        # 从训练这一步开始
.\run.ps1 -Stage 4 -Stop 4  # 只重跑解码评测
```

## 附录：报错急救表

| 报错关键词 | 原因 | 解决 |
| --- | --- | --- |
| `python 不是内部或外部命令` | 没装 Python 或没勾 PATH | 重装 Python，勾「Add to PATH」 |
| 终端没有 `(.venv)` | 虚拟环境没激活 | `.\.venv\Scripts\Activate.ps1` |
| `禁止运行脚本` | PowerShell 策略限制 | `Set-ExecutionPolicy -Scope CurrentUser RemoteSigned` |
| `未检测到 ffmpeg` | 没装/没重开终端 | 第 0.3 装好后**重开终端** |
| `No module named wenet.bin` | WeNet 训练代码没装 | 第 7 部分 `git clone` + `pip install -e .` |
| TTS 网络超时 | 没联网/被墙 | 检查网络后重跑第 4 步 |
| 训练报 unexpected key | WeNet 版本字段差异 | 按报错删/改 yaml 对应行 |
| 训练巨慢 | CPU + 参数偏大 | 调小 yaml 的 max_epoch / num_blocks |
| CER 很高 | 合成数据 + 小模型 | 正常；调大 --repeat、max_epoch；报告里说明 |
| 中文输出乱码 | Windows GBK 控制台 | 脚本已加 UTF-8 修复；若仍乱码可忽略，不影响功能 |
