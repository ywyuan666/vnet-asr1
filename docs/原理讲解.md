## 1. 任务与端到端 ASR

**任务**：输入一段语音波形（用户对安克耳机说的指令），输出对应的中文文字，
例如 语音「打开降噪」→ 文字「打开降噪」，进而触发耳机功能。

**传统方案 vs 端到端**：
- 传统 ASR = 声学模型(HMM-GMM/DNN) + 发音词典 + 语言模型，模块多、调试复杂。
- **端到端 ASR**：一个神经网络直接「语音 → 文字」，本项目即端到端。

**整体数据流**（系统框图）：

```
语音波形 16kHz
   │  分帧加窗 + 梅尔滤波
   ▼
Fbank 特征 [T, 80]
   │  Conv2d 下采样 4×
   ▼
Conformer 编码器  ──► 高层声学表示 memory [T/4, D]
   │                         │
   ├──► CTC 分支            ├──► 双向注意力解码器(L2R + R2L)
   │   (快速一遍解码)         │     (自回归生成文字)
   ▼                         ▼
       Attention Rescoring（CTC 出 N-best，注意力解码器重打分）
   ▼
识别文字「打开降噪」
```

## 2. 声学特征：Fbank

- 把波形按 **25ms 帧长、10ms 帧移** 分帧，每帧做傅里叶变换得到频谱，
  再经过 **80 个梅尔滤波器** → 每帧 80 维特征，得到 `[T, 80]` 的特征序列。
- 项目中由 WeNet 的 `fbank_conf` 在线计算（见 `conf/train_u2pp_conformer.yaml`）。
- **CMVN**：对特征做零均值单位方差归一化（`tools/make_cmvn.py`），让训练更稳。

---

## 3. Conformer 编码器（核心创新点之一）

Conformer = **Transformer 的全局自注意力** + **CNN 的局部建模能力**，二者结合。

一个 Conformer 块的结构（macaron「三明治」结构）：

```
输入
 ├─ ½ × 前馈网络(FFN)            ← 半步残差
 ├─ 多头自注意力(MHSA)           ← 捕捉全局、长距离依赖
 ├─ 卷积模块(Conv Module)        ← 捕捉局部、相邻帧关系
 ├─ ½ × 前馈网络(FFN)            ← 半步残差
 └─ LayerNorm
输出
```

- **多头自注意力**：让每一帧都能"看到"整句话，建模长距离上下文。
- **卷积模块**：逐点卷积 + GLU 门控 + 深度可分离卷积，专门抓局部细节（音素过渡）。
- 本项目对应代码：`model/u2pp_conformer_min.py` 里的 `ConformerBlock`。
- WeNet 配置：`encoder: conformer`，`num_blocks` 控制层数。

## 4. CTC 分支（Connectionist Temporal Classification）

- 问题：语音帧数 T 远多于文字数 L，且不知道每个字对应哪几帧（对齐未知）。
- **CTC** 引入一个特殊的 **空白符 `<blank>`**，允许网络对每一帧输出「某个字」或「空白」，
  再通过「去重 + 去空白」把帧级输出折叠成文字。它会自动学到对齐，无需人工标注。
- 例：帧级输出 `下下_一一__首` → 折叠 → `下一首`（`_` 是空白）。
- 优点：**解码快**（一遍、非自回归）。缺点：假设帧间条件独立，精度略逊注意力。
- 对应代码：`U2ppConformer` 里的 `ctc_fc` + `F.ctc_loss`。

---

## 5. 注意力解码器（Attention Decoder）

- 自回归地一个字一个字生成：`<sos>` → 下 → 一 → 首 → `<eos>`。
- 每生成一个字，都通过**交叉注意力**去"看"编码器输出 memory，决定关注哪几帧。
- 优点：**精度高**（建模了字与字之间的依赖）。缺点：必须串行生成，**慢**。

---

## 6. U2++：双向解码器 + 流式非流式统一（项目最大亮点）

U2++ = **U**nified **2**-pass ++，是 WeNet 的旗舰架构，在 U2 基础上升级：

### 6.1 两遍解码（Two-pass）
- **第一遍**：CTC 分支快速产生 N 条候选（N-best）。
- **第二遍（Rescoring）**：用注意力解码器对这 N 条候选重新打分，选最优。
- 兼顾「CTC 的快」与「注意力的准」。

### 6.2 双向注意力解码器（“++” 的来历）
- U2 只有 **左→右(L2R)** 一个解码器。
- U2++ 额外加一个 **右→左(R2L)** 解码器，把文字反过来也学一遍。
- 重打分时同时用 L2R 和 R2L 的分数（`reverse_weight` 控制 R2L 占比），
  对**句尾/边界**的识别更鲁棒，CER 进一步降低。
- 对应：WeNet 配置 `decoder: bitransformer` + `r_num_blocks`；
  教学代码 `decoder_l2r` 与 `decoder_r2l`。

### 6.3 流式 / 非流式统一
- 通过**动态 chunk 训练**（`use_dynamic_chunk: true`）+ **因果卷积**（`causal: true`），
  同一个模型既能**整句一次性识别**（非流式，最准），也能**边说边出字**（流式，低延迟）。
- 对耳机助手很实用：可以做到"说完一句立刻响应"。

### 6.4 联合损失函数
```
Loss = α · CTC_Loss
     + (1-α) · [ (1-β) · L2R注意力Loss + β · R2L注意力Loss ]
```
其中 α = `ctc_weight`(0.3)，β = `reverse_weight`(0.3)。
对应代码：`U2ppConformer.forward` 的 `loss` 计算。

## 7. 数据增强（提升小数据集表现）

- **速度扰动 speed perturb**：把音频按 0.9/1.0/1.1 倍速生成 3 个版本，数据量×3。
- **SpecAugment**：在 Fbank 特征上随机遮挡若干时间段/频率段，逼模型学更鲁棒的特征。
- 本项目还在数据合成阶段用了**多发音人 + 多语速**（`generate_anker_corpus.py`）。

## 8. 解码模式与评测指标

| 解码模式 | 说明 | 速度 | 精度 |
| --- | --- | --- | --- |
| `ctc_greedy_search` | CTC 贪心 | 最快 | 一般 |
| `ctc_prefix_beam_search` | CTC 前缀束搜索 | 快 | 较好 |
| `attention` | 纯注意力解码 | 慢 | 好 |
| `attention_rescoring` | CTC出候选+注意力重打分 | 中 | **最好** |

**评测指标 CER（字错误率）**：
```
CER = (替换 + 删除 + 插入) / 参考总字数 × 100%
```
越低越好。本项目 `recognize.py` 会自动计算并对比各模式。

---

## 9. 一句话总结（答辩可背）

> 本项目用 WeNet 框架实现了面向安克耳机的中文语音指令识别系统，
> 采用 U2++ 架构：以 Conformer 为编码器融合全局与局部特征，结合 CTC 与
> **双向注意力解码器**进行联合训练，并用 Attention Rescoring 两遍解码提升精度，
> 同时支持流式与非流式统一识别。
