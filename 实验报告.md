# Conformer + CTC/Attention/Transducer 三任务联合训练实验报告

## 1. 项目概述

本实验实现了一个基于 **Conformer 编码器 + CTC / Attention / Transducer 三解码头** 的端到端语音识别系统。参考 WeNet 架构设计，在一个统一的模型框架中同时训练三个解码器。

### 1.1 模型架构

```
Fbank(80-dim) → Conv2d下采样(4x) → PositionalEncoding → 6×ConformerBlock → Encoder Memory
     ├── CTC Linear → CTC Logits → CTC Loss (权重0.3)
     ├── Transformer Decoder(3层) → Attn Logits → CE Loss (权重0.3)
     └── Prediction LSTM(1层) + Joint Linear → RNNT Logits → RNN-T Loss (权重0.4)
```

### 1.2 关键参数

| 参数 | 值 | 说明 |
|------|-----|------|
| d_model | 144 | 模型维度 |
| n_head | 4 | 注意力头数 |
| d_ff | 1024 | 前馈网络隐藏维 |
| encoder_blocks | 6 | Conformer块数 |
| decoder_blocks | 3 | Attention解码器层数 |
| vocab_size | 35 | 词表大小(含blank/unk/sos/eos) |
| 总参数量 | ~6.6M | 模型总参数量 |

## 2. 数据集

### 2.1 数据生成

使用 Microsoft **edge-tts** 服务合成语音指令数据集：
- 12 条中文耳机语音指令（如"播放音乐"、"调高音量"、"接听电话"等）
- 5 种中文发音人（Xiaoxiao, Yunxi, Yunyang, Xiaoyi, Yunjian）
- 3 种语速（+0%, -10%, +12%）
- 5 轮重复 → 12 × 5 × 5 = **300 条音频**

数据分割：
- **训练集**: 240 条（每指令 20 条）
- **开发集**: 30 条
- **测试集**: 30 条

### 2.2 特征提取

- 16kHz 单声道 WAV
- 80 维 Fbank（帧长 25ms，帧移 10ms）
- 全局 CMVN 归一化

## 3. 训练过程

### 3.1 优化器与调度器

- **优化器**: Adam（betas=0.9, 0.98, eps=1e-9）
- **学习率调度**: Transformer 风格 WarmupLR
  - lr = d_model^(-0.5) × min(step^(-0.5), step × warmup_steps^(-1.5))
  - warmup_steps = 500

### 3.2 训练曲线

| Epoch | Train Loss | CV Loss | CTC(Train) | Attn(Train) | Trans(Train) |
|-------|------------|---------|------------|-------------|--------------|
| 1 | 42.40 | 31.62 | 24.69 | 3.28 | 85.01 |
| 10 | 5.34 | 4.99 | 4.86 | 0.68 | 9.58 |
| 30 | 1.85 | 2.15 | 2.14 | 0.53 | 2.58 |
| 60 | 0.12 | 0.16 | 0.17 | 0.05 | 0.14 |
| 100 | 0.04 | 0.09 | 0.04 | 0.02 | 0.18 |
| 157(最终) | 0.004 | 0.07 | 0.01 | 0.004 | 0.002 |

**损失下降趋势**：三任务的损失均随训练稳定下降，CV损失逐步接近训练损失，表明模型在有效学习。

### 3.3 关键修复：CMVN 归一化 Bug

**问题**：CMVN 脚本 `make_cmvn.py` 存储的是 `mean_stat`（特征累加和）和 `var_stat`（特征平方累加和），但在 `train.py` 和 `recognize.py` 中直接使用累加和作为均值参与归一化，导致特征值被放大了约 frame_num 倍。

**症状**：不同音频的 CMVN 归一化后特征余弦相似度 = 1.0000（完全相同），模型无法区分不同输入，所有解码器输出固定结果（CTC输出"调一话"、Attention输出"调低音量"、Transducer输出"调低"）。

**修复**：
```python
# 正确做法
mean = mean_stat / frame_num
var = var_stat / frame_num - mean * mean
feat_cmvn = (feat - mean) / sqrt(var + eps)
```

**修复后效果**：不同音频特征相似度从 1.0000 降至 0.67-0.70，模型能够正确区分不同语音指令。

## 4. 评测结果

使用测试集（30 条）评估三种解码模式：

| 解码模式 | CER | 精确匹配率 | 正确数/总数 |
|---------|-----|-----------|------------|
| **Attention** | **0.87%** | **96.67%** | **29/30** |
| CTC Greedy | 25.22% | 10.00% | 3/30 |
| Transducer Greedy | 9.57% | 76.67% | 23/30 |

### 4.1 各解码模式分析

**Attention 解码器（最佳）**：
- CER=0.87%，仅 1 个错误（"下一首" → "上一首"）
- 这两个命令仅首字不同（"下" vs "上"），在 TTS 合成中发音极为相似
- 自回归解码利用了完整的语言模型上下文信息

**CTC 贪心解码**：
- CER=25.22%，主要问题是字符重复（"播放音音"）和缺字（"播放音"、"接听话"）
- 贪心解码未考虑帧间的依赖关系，易产生重复/缺失错误
- 使用 beam search 可显著改善

**Transducer 解码**：
- CER=9.57%，主要问题是末尾字符丢失（"调低" → "调低音量"）
- 贪心 blank-skip 算法在末尾帧易提前触发 blank
- 需 beam search 或 length penalty 改善

## 5. 结论与展望

### 5.1 实验结论

1. **Conformer + CTC/Attention/Transducer 三任务联合训练是可行的**，在 300 条 TTS 合成数据上即达到 Attention CER=0.87% 的优秀效果。

2. **CMVN 的正确计算至关重要**，错误使用原始累加和导致特征完全丧失区分度。

3. **数据量是关键**：144 条数据时模型几乎不收敛，扩展到 300 条后效果显著提升。

4. **Attention 解码器在三者中表现最优**，自回归解码充分利用了语言模型信息。

### 5.2 改进方向

1. **增加数据集**：扩展到真实录音数据、更多指令、更多发音人
2. **Beam Search 解码**：替代贪心解码，尤其改善 CTC 和 Transducer
3. **模型优化**：增加 Encoder 深度（12 层）、使用更大 d_model（256）
4. **语言模型融合**：在 Attention 解码中融合外部语言模型
5. **SpecAugment**：训练时加入频谱增强，提高泛化能力
